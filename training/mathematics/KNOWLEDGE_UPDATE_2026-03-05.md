## 2. Time Series Anomaly Detection

**Source signals (2025, MDPI Applied Sciences + arXiv + Springer):**

- **Signal Processing + Deep Learning hybrid** (MDPI, Jun 2025): Two-step approach — first pass uses classical signal processing (filtering, decomposition) to structure the time series; second pass uses deep learning for anomaly scoring. Combines interpretability of DSP with pattern power of DL.

- **Reconstruction-based DL methods review** (Springer, Oct 2025): Comprehensive evaluation of autoencoder-family anomaly detectors on multivariate time series. Hybrid convolutional autoencoders (random + dilated convolutions) show best multi-scale temporal feature extraction. Key finding: no single method dominates; domain matters.

- **Adaptive Bottleneck + Dual Adversarial Decoder** (arXiv 2405.15273, Mar 2025): Generalist anomaly detector design. Adaptive bottleneck allows the model to tune its compression ratio per series. Dual adversarial decoders separate normal reconstruction from anomaly reconstruction — improves precision on point anomalies.

- **Industry Perspective on Open Challenges** (arXiv 2502.05392, Feb 2025): Benchmark confusion is a major problem — many published results are not reproducible across datasets. Recommends: (1) evaluation on multiple anomaly type categories, not aggregate F1; (2) explicit runtime + memory benchmarks; (3) distribution shift testing.

**Implications for Pythagoras work:**
- For supply chain time series (demand, lead time, inventory): signal processing preprocessing (STL decomposition, FFT filtering) before ML anomaly detection is now considered best practice
- When choosing a detection method, ask: point anomaly vs. contextual anomaly vs. collective anomaly — different methods optimize for different types
- Aggregate F1 is an insufficient evaluation metric; use precision/recall by anomaly type

## 3. Monte Carlo Simulation — Variance Reduction (2025)

**Source signals (2025, Dean Francis Press + INFORMS + Nature):**

- **Control Variates for Option Pricing** (Dean Francis Press, Jun 2025): Comparative analysis confirms multiple control variates (linear combination) outperform single control variate and antithetic variates for European/Power option pricing. Practical takeaway: use multivariate control variates when you have correlated reference variables.

- **Multifidelity Covariance Estimation via Manifold Regression** (SIAM, Jan 2025): New approach to estimating covariance matrices for MC input models. Works on the manifold of symmetric positive definite (SPD) matrices — more numerically stable than standard regression, especially in high-dimensional parameter spaces. Relevant for simulation input modeling.

- **DeGVR Method** (Nature Research Intelligence, 2025): Density-extrapolation Global Variance Reduction — dual-material density simulation approach for deep-penetration problems. Domain-specific (radiation transport) but the extrapolation idea (run cheaper approximation, extrapolate to full fidelity) generalizes to supply chain simulation.

**Implications for Pythagoras work:**
- Preferred variance reduction stack for SC Monte Carlo: stratified sampling + control variates (use historical demand mean as CV) + antithetic variates for symmetric distributions
- For high-dimensional sensitivity analysis: consider multifidelity approach — run cheap model extensively, run expensive model selectively, combine estimates
- Document variance reduction method chosen and achieved variance reduction ratio in every simulation deliverable

## 4. Vector Embeddings & Similarity Search (2025)

**Source signals (arXiv, Pinecone SIGIR '25, meilisearch):**

- **GleanVec** (arXiv 2410.22347): Piecewise linear nonlinear dimensionality reduction for vector search. Improves over linear methods (PCA, IVF) in search accuracy at minimal performance cost. Uses "LeanVec-Sphering" for efficient similarity computation in compressed space. Relevant for our hive memory / LanceDB deployment.

- **DIME framework** (Pinecone / SIGIR '25, Jul 2025): Reproducibility and generalizability evaluation framework for embedding-based retrieval. Key finding: many published embedding benchmarks don't generalize across domains. DIME proposes cross-domain evaluation protocol. Takeaway: when selecting an embedding model for KB retrieval, test on domain-specific queries, not generic benchmarks.

- **HNSW remains dominant** (meilisearch 2025 guide): Hierarchical Navigable Small-World graphs are still the best-practice index structure for approximate nearest neighbor search at scale. Embedding drift over time (as language/domain evolves) is a recognized production problem — recommends periodic re-indexing.

**Implications for Pythagoras work:**
- Hive memory search (LanceDB + embeddings): embedding drift is a real risk as team knowledge base grows. Flag for periodic re-embedding review (quarterly).
- When building a KB retrieval system, prefer domain-specific evaluation over generic benchmarks
- For high-dimensional KB search: piecewise linear compression (GleanVec-style) worth considering if search latency becomes an issue
